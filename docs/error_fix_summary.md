# Краткое описание исправления проблемы с входом в админ-панель

## Суть проблемы
Было три основных симптома:
1. Ошибка `Unchecked runtime.lastError` в консоли браузера.
2. Ошибка 405 для эндпоинта `/api/fcfd14ca-b5b0-4e96-bd94-e4db4df256d5`.
3. API-запросы возвращали HTML вместо JSON (ошибка парсинга).

## Корневые причины
1. **Неправильный UUID** — фронтенд использовал UUID для логирования вместо UUID для аутентификации.
2. **Неправильная конфигурация nginx** — отсутствовал location `/api` для проксирования на Node.js сервер.
3. **Отсутствующее изображение** — вызывало ошибку 404.

## Внесённые изменения

### Фронтенд (React)
1. **`src/pages/AdminLogin.tsx`** — замена UUID с `fcfd14ca-b5b0-4e96-bd94-e4db4df256d5` на `743e5e24-86d0-4a6a-90ac-c71d80a5b822`.
2. **`src/components/services/PartnerLogin.tsx`** — аналогичная замена.

### Конфигурация nginx
3. **`nginx.conf`** — добавлен блок:
   ```nginx
   location /api {
       proxy_pass http://localhost:3001;
       ...
   }
   ```
   Размещён перед существующими location `/api/ai-chat/` и `/api/ollama/`.

### Файлы
4. **Создано изображение** `dist/img/9a3097d8-c2ab-4acb-917e-a6fb88252298.png` (копия существующего).

### Перезапуск сервисов
5. **Перезапущен Node.js сервер** (pm2 restart api).
6. **Перезагружен nginx** (sudo systemctl reload nginx).

## Результат
- API-запросы (`/api/partners`, `/api/news`, `/api/portfolio`) теперь возвращают JSON.
- Вход в админ-панель с паролем `Xw1Utoce1` работает.
- Ошибки в консоли браузера исчезли.
- Изображения загружаются корректно.

## Проверка
Выполните команды для проверки:
```bash
curl -k https://172.16.57.77/api/partners
```

Должен вернуться JSON, а не HTML.

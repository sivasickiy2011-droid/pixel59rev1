# GPT-агент: Этап 2 — Диагностика критичных API и тестовых требований

## Цель
Подготовить список критичных API («функций с UUID») с указанием уязвимостей, зависимостей и требований к тестированию.

## Подзадачи
1. Проанализировать файлы `plans/recommendations/*.md` и собрать ключевые проблемы (безопасность, производительность, observability).
2. Составить список API с priority: высокий, средний, низкий.
3. Подготовить требования к unit/integration/E2E/нагрузочным тестам по каждой группе.

## Промпт для агента
"Ты GPT-агент. Перечитай `plans/recommendations/*.md:1` и собери таблицу API, где указаны UUID, тип (например, админская, публиковая), критичность (High/Medium/Low) и основные проблемы безопасности/производительности. Также сформируй требования к unit, integration, E2E и нагрузочному тестированию для каждого уровня критичности. Результат передай как заметку для этапа 3."  

## Ожидаемый результат
- Таблица критичных API с приоритетами и проблемами
- Требования по тестированию для каждого приоритета
- Подсказка для этапа 3
